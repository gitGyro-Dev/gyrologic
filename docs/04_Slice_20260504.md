# 4. Slice / slice-ing / slice-done の正式定義

## 4.1 結論

ここでは、`Slice` を一枚岩で扱わず、次の三層に分ける。

```text
Slice
= Structureを現す作用概念全体
```

```text
slice-ing
= Sliceが進行している時間ありの作用過程
```

```text
slice-done
= Sliceが完了した時間なしの成立結果
```

この整理により、Gyro Logic の中核である：

```text
Structure → Slice → Stability
```

を時間なしの最小構造として保ちながら、Gyro Process / Gyro Loop における時間ありの作用過程も扱えるようになる。

---

## 4.2 なぜ Slice を三層に分けるのか

これまで `Slice` と呼んでいたものには、実は二つの側面がある。

ひとつは、理論上の概念としての Slice である。

```text
Structureを、ある見方・角度・粒度・文脈で現す作用概念
```

もうひとつは、実際に Slice が進行して、完了結果に到達するまでの過程である。

```text
Structureを切り出している途中の時間ありの過程
```

さらに、その過程が終わった結果がある。

```text
Sliceが完了した成立結果
```

この三つを区別しないと、次の混同が起きる。

```text
Structure → Slice → Stability に時間があるのか
SliceはStabilityへ向かうのか
計算中の状態はどこに属するのか
Stabilityへ渡るのは何か
```

そのため、次のように分ける。

```text
Slice / slice-ing / slice-done
```

---

## 4.3 Slice の正式定義

```text
Sliceとは、
Structureをある見方・角度・粒度・文脈で現す作用概念全体である。
```

英語では：

```text
Slice is the act-concept by which a Structure appears under a certain perspective, angle, granularity, or context.
```

ここで重要なのは、Slice は「概念全体」だということである。

つまり、Slice という語は、広義には次の二つを含む。

```text
slice-ing
slice-done
```

ただし、厳密に議論する場合は、両者を分ける。

---

## 4.4 Slice は Gyro Unit に属する

Gyro Unit では、Slice は時間なしの構造要素として現れる。

```text
Gyro Unit:
Structure → Slice → Stability
```

ここでの Slice は、時間過程ではなく、論理的な関係の中にある概念である。

つまり：

```text
StructureがSliceとして現れる
```

という成立関係を表す。

この段階では、slice-ing の進行時間は明示しない。

---

## 4.5 slice-ing の正式定義

```text
slice-ingとは、
Sliceが進行している時間ありの作用過程である。
```

英語では：

```text
slice-ing is the time-including act-process in which Slice is being performed.
```

slice-ing は、Gyro Process に属する。

```text
Gyro Process:
Structure → Operator Orientation → slice-ing → slice-done → Stability → Operator Response
```

### 4.5.1 slice-ing に含まれるもの

slice-ing には、次のような時間が含まれる。

```text
観測時間
探索時間
計算時間
変換時間
選別時間
比較時間
圧縮時間
```

つまり、Gyro 的には：

```text
計算時間 = slice-ing の時間
```

である。

式や関係そのものが時間を持つのではなく、slice-done に到達するまでの作用過程が時間を持つ。

### 4.5.2 slice-ing の時間表現

slice-ing は、時間幅を持つ。

```text
slice-ing(t0〜t1)
```

これは、時刻 `t0` から `t1` まで、Slice が進行していることを表す。

```text
S(t0)
→ slice-ing(t0〜t1)
→ slice-done(t1)
```

---

## 4.6 slice-done の正式定義

```text
slice-doneとは、
Sliceが完了した成立結果であり、Stabilityへ渡される表現状態である。
```

英語では：

```text
slice-done is the completed result of Slice, treated as the established representation state passed to Stability.
```

slice-done は、slice-ing の結果として現れる。

```text
slice-ing
→ slice-done
```

そして、Stability はこの slice-done に現れる。

```text
slice-done
→ Stability
```

### 4.6.1 slice-done は時間なしとして扱える

slice-done は、slice-ing の完了時点に得られる結果である。

時間つきで書けば：

```text
slice-done(t1)
```

である。

しかし、slice-done そのものは「完了した成立結果」であり、Gyro Unit 側では時間なしの構造要素として扱える。

つまり：

```text
slice-ing = 時間ありの過程
slice-done = 時間なしの成立結果
```

である。

### 4.6.2 slice-done の内容

slice-done は、次のように表せる。

```text
slice-done = X + Δ
```

ここで：

```text
X = Sliceによって現れたRepresentation
Δ = StructureとRepresentationのズレ
```

つまり、slice-done は完全な Structure そのものではない。

```text
slice-done = Representation + Deviation
```

であり、ここに Gyro Logic のズレ構造が現れる。

---

## 4.7 Stability に渡るのは何か

Stability に渡るのは、基本的に `slice-done` である。

```text
Structure
→ slice-ing
→ slice-done
→ Stability
```

つまり：

```text
Stability = Stability(slice-done)
```

または：

```text
σ = Stab(X, Δ)
```

である。

重要なのは、Stability は slice-ing 中の過程を直接評価するのではなく、基本的には slice-done の結果に現れるという点である。

ただし、Loop 内では、複数の slice-done を比較することで Stability over time が扱われる。

---

## 4.8 Operator Orientation との関係

Operator Orientation は slice-ing の前にある。

```text
Structure
→ Operator Orientation
→ slice-ing
→ slice-done
```

つまり、Orientation は Slice の方向づけだが、slice-ing そのものではない。

```text
Operator Orientation = Slice前の指向性
slice-ing = Sliceが進行している過程
slice-done = Sliceの完了結果
```

したがって：

```text
OrientationがSliceを方向づける
```

とは言えるが、

```text
Orientation = Slice
```

ではない。

---

## 4.9 Slice は Stability へ目的的に向かわない

ここは非常に重要である。

誤りは次である。

```text
SliceがStabilityへ向かう
```

正しくは：

```text
SliceはStructureを現す作用であり、
Stabilityはslice-doneの結果に現れる。
```

さらに厳密には：

```text
Operator OrientationがSliceを方向づける
slice-ingが進行する
slice-doneが成立する
その結果としてStabilityが現れる
```

である。

---

## 4.10 数式風表現

Gyro Unit では：

```text
X + Δ = O(S)
σ = Stab(X, Δ)
```

ここでは時間を明示しない。

Gyro Process では：

```text
S(t0)
B(t0)
O_B : S(t0) ⇝ X(t1) + Δ(t1)
σ(t1) = Stab(X(t1), Δ(t1))
```

ここで：

```text
B(t0) = Operator Orientation
O_B = Orientation B によって方向づけられた Slice
⇝ = 時間ありの slice-ing
X(t1) + Δ(t1) = slice-done
σ(t1) = Stability
```

この `⇝` を使うことで、時間ありの slice-ing と、成立結果としての slice-done を区別できる。

---

## 4.11 三層の対応表

| 概念 | 定義 | 時間 | 所属 |
|---|---|---|---|
| Slice | Structureを現す作用概念全体 | 原則として時間を明示しない | Gyro Unit |
| slice-ing | Sliceが進行している作用過程 | 時間あり | Gyro Process |
| slice-done | Sliceが完了した成立結果 | 時間なしとして扱える | Gyro Unit / Gyro Process の接点 |

---

## 4.12 誤解防止

### 誤り1

```text
Slice = slice-ing
```

これは不正確である。

正しくは：

```text
Sliceは概念全体であり、
slice-ingはその時間ありの作用過程である。
```

### 誤り2

```text
Slice = slice-done
```

これも不正確である。

正しくは：

```text
slice-doneはSliceが完了した成立結果である。
```

### 誤り3

```text
Stabilityはslice-ingを評価する
```

これは不正確である。

正しくは：

```text
Stabilityは基本的にslice-doneに現れる状態量である。
```

### 誤り4

```text
slice-doneには時間が流れている
```

これは不正確である。

正しくは：

```text
slice-doneは完了した成立結果であり、
時間なしの状態として扱える。
```

### 誤り5

```text
計算結果そのものに時間がある
```

これは不正確である。

正しくは：

```text
時間があるのは、計算結果に至るslice-ingの過程である。
```

## 4.13 正式定義として使える文章

### English

```text
Slice is the act-concept by which a Structure appears under a certain perspective, angle, granularity, or context.

For precise theoretical treatment, Slice is divided into slice-ing and slice-done.

slice-ing is the time-including act-process in which Slice is being performed.
It includes the temporal aspects of observation, computation, search, transformation, comparison, and compression.

slice-done is the completed result of Slice.
It is the established representation state passed to Stability.
It may be represented as:

slice-done = X + Δ

where X denotes the representation produced by Slice, and Δ denotes the deviation between Structure and Representation.

Stability appears in slice-done.
Therefore, Stability is not the evaluator of Slice, but the state quantity that appears in the completed result of Slice.
```

### 日本語

```text
Sliceとは、Structureをある見方・角度・粒度・文脈で現す作用概念全体である。

厳密な理論整理において、Sliceは slice-ing と slice-done に分けられる。

slice-ingとは、Sliceが進行している時間ありの作用過程である。
そこには、観測、計算、探索、変換、比較、圧縮などの時間的側面が含まれる。

slice-doneとは、Sliceが完了した成立結果である。
それはStabilityへ渡される表現状態である。
slice-doneは次のように表せる。

slice-done = X + Δ

ここで、XはSliceによって現れたRepresentationを表し、ΔはStructureとRepresentationのズレを表す。

Stabilityはslice-doneに現れる。
したがって、StabilityはSliceを評価する主体ではなく、Sliceが完了した結果に現れる状態量である。
```

## 4.14 最終固定文

この段階では、次を固定してよい。

```text
Slice = Structureを現す作用概念全体
```

```text
slice-ing = Sliceが進行している時間ありの作用過程
```

```text
slice-done = Sliceが完了した時間なしの成立結果
```

```text
Stabilityに渡るのは、基本的にslice-doneである。
```

```text
時間があるのは、Sliceの成立結果ではなく、slice-doneへ至るslice-ingの過程である。
```

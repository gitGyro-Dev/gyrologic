# 5. Stability as property / Stability over time の区別

## 5.1 結論

Stability は、まず次の二層に分けるのがよい。

```text
Stability as property
= 単一の slice-done に現れる安定性の状態量
= 時間なし
```

```text
Stability over time
= 複数の slice-done / Process をまたいで見える安定性
= 時間あり
```

つまり、Stability そのものを「時間あり」と決めるのではなく、どの層で扱っているかによって分ける。

```text
Gyro Unit 内の Stability
= Stability as property
= 時間なし
```

```text
Gyro Loop 内の Stability
= Stability over time
= 時間あり
```

---

## 5.2 前提：Stability は評価者ではない

最初に固定するべき原則がある。

```text
Stabilityは評価者ではない。
Stabilityは、Slice結果に現れる安定性の状態量である。
```

これは、今回の二層化でも変わらない。

誤りは次である。

```text
Stabilityが評価する
Stabilityが判断する
Stabilityが次のSliceを決める
StabilityがLoopを止める
```

正しくは：

```text
Operator ResponseがStabilityを受けて反応する
```

である。

Stability は、あくまで Response の入力になる状態量である。

---

## 5.3 Stability as property

### 定義

```text
Stability as propertyとは、
単一のslice-doneに現れる安定性の状態量である。
```

英語では：

```text
Stability as property is the state quantity of stability that appears in a single slice-done.
```

これは Gyro Unit に対応する。

```text
Structure → Slice → Stability
```

または、slice-done を明示すれば：

```text
Structure → slice-done → Stability
```

である。

### 5.3.1 時間なしとして扱える理由

Stability as property は、単一の slice-done に現れる状態量である。

```text
slice-done → Stability
```

これは、成立済みの結果に対する状態量なので、時間過程を明示しない。

たとえば：

```text
X + Δ
```

という slice-done が得られたとき、

```text
σ = Stab(X, Δ)
```

として Stability が現れる。

ここでは、`X + Δ` に到達するまでの時間は扱わない。

時間があるのは：

```text
slice-ing
```

である。

Stability as property は、slice-done の結果に現れる状態量として扱う。

### 5.3.2 数式風表現

```text
X + Δ = O(S)
σ = Stab(X, Δ)
```

ここで：

```text
S = Structure
O(S) = Slice結果
X = Representation
Δ = Deviation
σ = Stability as property
```

この表現では、時間を明示しない。

つまり：

```text
σ
```

は、単一の Slice 結果に対して現れる安定性の状態量である。

### 5.3.3 Stability as property の役割

Stability as property は、次を表す。

```text
Slice結果がどれだけ安定しているか
Representationがどれだけ保持可能か
Δがどれだけ許容可能か
意味・同一性・判断がどれだけ成立しやすいか
```

ただし、Stability as property はそれ自体で判断しない。

```text
σが高いから続ける
```

ではなく：

```text
Operator Responseがσを受けてContinueを選ぶ
```

である。

---

## 5.4 Stability over time

### 定義

```text
Stability over timeとは、
複数のslice-doneまたはGyro Processをまたいで観測される安定性の系列・持続・変化である。
```

英語では：

```text
Stability over time is the persistence, variation, or trajectory of stability observed across multiple slice-done results or Gyro Processes.
```

これは Gyro Loop に対応する。

```text
Process₀ → Process₁ → Process₂ → ...
```

各 Process において Stability が現れる。

```text
σ₀, σ₁, σ₂, ..., σₙ
```

この系列が Stability over time である。

### 5.4.1 時間ありとして扱う理由

Stability over time は、単一の slice-done ではなく、複数の slice-done の系列を扱う。

```text
slice-done₀
slice-done₁
slice-done₂
...
```

それぞれに Stability が現れる。

```text
σ₀
σ₁
σ₂
...
```

この変化・持続・収束・発散を見るためには、時間が必要である。

したがって：

```text
Stability over time = 時間あり
```

である。

### 5.4.2 数式風表現

```text
σ_n = Stab(X_n, Δ_n)
```

Loop 内では：

```text
{σ_n}_{n=0}^{N}
```

または：

```text
σ(t₀), σ(t₁), σ(t₂), ..., σ(tₙ)
```

として表せる。

この系列から、次のような状態を見る。

```text
持続している
収束している
発散している
振動している
崩壊している
過剰に固定されている
Jumpが必要になっている
```

---

## 5.5 Stability as property と Stability over time の関係

この二つは別概念ではなく、同じ Stability を異なる層で見たものである。

```text
Stability as property
= 単発のSlice結果に現れるStability
```

```text
Stability over time
= 単発Stabilityの系列として見えるStability
```

つまり：

```text
Stability over time
= sequence of Stability as property
```

または：

```text
Stability over time = {σ₀, σ₁, σ₂, ..., σₙ}
```

である。

---

## 5.6 Gyro Unit / Process / Loop との対応

| 層 | Stability の扱い | 時間 |
|---|---|---|
| Gyro Unit | Stability as property | 時間なし |
| Gyro Process | slice-done時点に現れるStability | 局所的に時間あり |
| Gyro Loop | Stability over time | 時間あり |

もう少し細かく書くと：

```text
Gyro Unit:
Structure → Slice → Stability
= σ
```

```text
Gyro Process:
Structure → Orientation → slice-ing → slice-done → Stability → Response
= σ(t₁)
```

```text
Gyro Loop:
Process₀ → Process₁ → Process₂ → ...
= σ₀, σ₁, σ₂, ...
```

---

## 5.7 Stability と Response の関係

Stability は Response の入力である。

```text
Stability → Operator Response
```

ただし、より正確には：

```text
Operator receives Stability
↓
Operator Response occurs
```

である。

形式的には：

```text
R_n = Response(σ_n, Δ_n)
```

ここで：

```text
σ_n = Stability
Δ_n = Deviation
R_n = Operator Response
```

重要なのは、次である。

```text
σ_n が Response するのではない。
Operator が σ_n を受けて Response する。
```

---

## 5.8 Stability と Δ の関係

Stability は、Δ と深く関係する。

単純化すれば：

```text
Δが小さいほどStabilityは高くなりやすい
Δが大きいほどStabilityは低くなりやすい
```

ただし、これは絶対ではない。

Gyro Logic では、Δ は単なる誤差ではなく、Structure と Representation のズレである。

場合によっては、小さな Δ が停滞を生み、大きな Δ が Jump の材料になる可能性もある。

したがって、Stability は単純な「ズレの小ささ」だけではない。

```text
Stability = Δを含むSlice結果の保持可能性
```

と考える方が安全である。

---

## 5.9 Stability over time と収束

Stability over time を見ると、収束が扱える。

```text
σ₀ → σ₁ → σ₂ → ... → σ*
```

このとき：

```text
σ*
```

は、安定した attractor 的状態として扱える。

ただし、ここでも注意がある。

```text
Stabilityが収束する
```

というより：

```text
Gyro Loopにおいて、Stability系列が収束的に見える
```

と書く方が正確である。

主体は Stability ではなく、Loop の系列である。

---

## 5.10 Stability over time と崩壊

逆に、Stability over time が崩れる場合もある。

```text
σ₀ → σ₁ → σ₂ → ... → collapse
```

これは、次のような状態である。

```text
Stabilityが低下し続ける
Δが増大する
Representationが保持できない
Orientationが不適切
SliceがStructureを捉えられない
Voidが現れる
```

この場合、Operator Response は次を選ぶ可能性がある。

```text
Adjust Slice
Update Orientation
Update Structure
Jump
Stop
```

---

## 5.11 Stability over time と過剰安定

Stability は高ければよい、とは限らない。

Loop において、過剰に安定している状態がある。

```text
σ₀ ≈ σ₁ ≈ σ₂ ≈ ... ≈ σₙ
```

これは一見よい状態に見える。

しかし、Structure が変化しているにもかかわらず Representation が固定されている場合、それは硬直化かもしれない。

つまり：

```text
High stability can mean robustness.
High stability can also mean rigidity.
```

日本語では：

```text
高いStabilityは頑健性を意味する場合もある。
しかし、過剰なStabilityは硬直化を意味する場合もある。
```

ここでも判断するのは Stability ではなく、Operator Response である。

---

## 5.12 Stability over time と Jump

Jump は、Stability over time の中で重要になる。

たとえば：

```text
σ₀, σ₁, σ₂
```

が低下し続ける場合、あるいは未定義になる場合：

```text
σ_n = undefined
```

既存の Slice / Orientation / Structure では対応できない可能性がある。

このとき、Operator Response は Jump を選ぶ。

```text
Void / large Δ / unstable σ-series
↓
Operator Response
↓
Jump
```

ここでも：

```text
StabilityがJumpする
```

ではない。

```text
Operator ResponseがStability系列を受けてJumpを選ぶ
```

である。

---

## 5.13 Stability as property / Stability over time の対応表

| 概念 | 定義 | 時間 | 所属 | 表現 |
|---|---|---|---|---|
| Stability as property | 単一のslice-doneに現れる安定性 | 時間なし | Gyro Unit | σ |
| Stability in Process | Process内でslice-done後に現れる安定性 | t₁に現れる | Gyro Process | σ(t₁) |
| Stability over time | 複数ProcessにまたがるStability系列 | 時間あり | Gyro Loop | {σₙ} |

---

## 5.14 誤解防止

### 誤り1

```text
Stabilityは常に時間ありである
```

これは不正確である。

正しくは：

```text
単発のStabilityは時間なしの状態量として扱える。
Stability over timeは時間ありである。
```

### 誤り2

```text
Stabilityは常に時間なしである
```

これも不正確である。

正しくは：

```text
Gyro Unit内では時間なしだが、Gyro Loop内では時間をまたいだ系列として扱える。
```

### 誤り3

```text
StabilityがLoopを制御する
```

これは不正確である。

正しくは：

```text
Operator ResponseがStabilityを受けてLoopを制御する。
```

### 誤り4

```text
Stabilityが評価者である
```

これは不正確である。

正しくは：

```text
Stabilityは評価者ではなく、評価される状態量である。
```

### 誤り5

```text
高Stabilityなら必ず良い
```

これは不正確である。

正しくは：

```text
高Stabilityは頑健性を意味する場合もあるが、硬直化を意味する場合もある。
```

## 5.15 正式定義として使える文章

### English

```text
Stability should be distinguished into two layers: Stability as property and Stability over time.

Stability as property is the state quantity of stability that appears in a single slice-done. It belongs to the time-free layer of Gyro Unit and may be represented as:

σ = Stab(X, Δ)

where X denotes the representation produced by Slice and Δ denotes the deviation between Structure and Representation.

Stability over time is the persistence, variation, or trajectory of stability observed across multiple slice-done results or Gyro Processes. It belongs to the time-including layer of Gyro Loop and may be represented as:

{σ_n}_{n=0}^{N}

Stability is not an evaluator. It does not decide the next Slice, Structure, continuation, stopping, or Jump. Rather, Stability is received by the Operator, and Operator Response determines the next action in the Loop.
```

### 日本語

```text
Stabilityは、Stability as property と Stability over time の二層に分けて扱うべきである。

Stability as propertyとは、単一のslice-doneに現れる安定性の状態量である。
これはGyro Unitの時間なしの層に属し、次のように表せる。

σ = Stab(X, Δ)

ここで、XはSliceによって現れたRepresentationを表し、ΔはStructureとRepresentationのズレを表す。

Stability over timeとは、複数のslice-doneまたはGyro Processをまたいで観測されるStabilityの持続・変化・軌跡である。
これはGyro Loopの時間ありの層に属し、次のように表せる。

{σ_n}_{n=0}^{N}

Stabilityは評価者ではない。
Stabilityは、次のSlice、Structure、継続、停止、Jumpを決めない。
StabilityはOperatorに受け取られ、Operator ResponseがLoopにおける次の作用を決める。
```

## 5.16 最終固定文

この段階では、次を固定してよい。

```text
Stability as property
= 単一のslice-doneに現れる時間なしの状態量
```

```text
Stability over time
= 複数のslice-done / Processをまたいで観測される時間ありのStability系列
```

```text
Stabilityは評価者ではなく、評価される状態量である。
```

```text
StabilityはLoopを制御しない。
Operator ResponseがStabilityを受けてLoopを制御する。
```

```text
高いStabilityは頑健性を意味する場合もあるが、過剰なStabilityは硬直化を意味する場合もある。
```

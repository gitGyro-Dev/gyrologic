# Gyro Loop = Processの反復構造

## 3.1 結論

Gyro Loop は、Gyro Process の反復構造である。

```text
Gyro Unit
= Structure → Slice → Stability
= 時間なしの最小論理構造
```

```text
Gyro Process
= Structure → Operator Orientation → slice-ing → slice-done → Stability → Operator Response
= 時間ありの一周期
```

```text
Gyro Loop
= Gyro Process の反復構造
```

より具体的には、Gyro Loop は次のように定義される。

```text
Gyro Loop
= Gyro Process が Operator Response によって
  Next Structure / Next Slice / Stop / Continue / Jump
  へ接続される反復構造
```

---

## 3.2 Gyro Loop の定義

```text
Gyro Loopとは、
Gyro ProcessがOperator Responseを通じて、
次のStructure、次のSlice、継続、停止、Jumpへ接続されることで成立する反復構造である。
```

英語では：

```text
A Gyro Loop is the iterative structure formed when a Gyro Process is connected, through Operator Response, to the next Structure, next Slice, continuation, stopping, or Jump.
```

---

## 3.3 Gyro Loop の基本構造

```text
Structure
↓
Operator Orientation
↓
slice-ing
↓
slice-done
↓
Stability
↓
Operator Response
↓
Next Structure / Next Slice / Stop / Continue / Jump
↓
Loop
```

または、より簡潔には：

```text
Gyro Process
↓
Operator Response
↓
Next Process / Stop / Jump
```

---

## 3.4 Unit / Process / Loop の関係

```text
Gyro Unit
= 時間なしの最小構造

Gyro Process
= 時間ありの一周期

Gyro Loop
= Processの反復構造
```

対応関係は次の通りである。

```text
Gyro Unit:
Structure → Slice → Stability

Gyro Process:
Structure → Operator Orientation → slice-ing → slice-done → Stability → Operator Response

Gyro Loop:
Gyro Process_n → Gyro Process_{n+1} → Gyro Process_{n+2} → ...
```

つまり、Gyro Loop は Gyro Unit を直接反復するというより、Gyro Process を反復する構造である。

---

## 3.5 時間構造

Gyro Loop は、明確に時間ありである。

Gyro Process 一回分を次のように表す。

```text
P_n:
S_n(t0)
→ B_n(t0)
→ slice-ing_n(t0〜t1)
→ slice-done_n(t1)
→ σ_n(t1)
→ R_n(t1〜t2)
```

Gyro Loop は次のように表せる。

```text
P_0 → P_1 → P_2 → ... → P_n
```

より具体的には：

```text
S_n(t0)
→ B_n(t0)
→ slice-ing_n(t0〜t1)
→ X_n(t1) + Δ_n(t1)
→ σ_n(t1)
→ R_n(t1〜t2)
→ S_{n+1}(t2) / B_{n+1}(t2) / O_{n+1}(t2) / Stop / Jump
```

ここで、次の Process が始まる。

```text
P_{n+1}
```

---

## 3.6 Loop Decision

Gyro Loop における重要点は、Loop Decision である。

ただし、ここでも注意が必要である。

```text
Stabilityが次を決める
```

のではない。

正しくは：

```text
Operator ResponseがStabilityを受けて次を決める
```

である。

Loop Decision は、Operator Response の結果として現れる。

```text
D_n = Decision produced by Operator Response
```

Decision の候補は次である。

```text
D_n ∈ {
  Continue,
  Stop,
  Adjust Slice,
  Update Structure,
  Update Orientation,
  Jump
}
```

---

## 3.7 Continue

```text
Continue
= 現在の構造・方向性・Slice方針を大きく変えずに、次のProcessへ進むこと
```

これは、Stability が十分に保たれ、Δ が許容範囲にある場合に選ばれやすい Response である。

ただし、Continue を決めるのは Stability ではない。

```text
Operator ResponseがStabilityを受けてContinueを選ぶ
```

---

## 3.8 Stop

```text
Stop
= Loopを終了すること
```

Stop は、次のような場合に起こる。

```text
目的上十分なStabilityが得られた
これ以上のSliceが不要になった
Loop継続の意味が消えた
Operatorが停止を選んだ
```

ここでも、Stability が止めるのではない。

```text
Operator ResponseがStopを選ぶ
```

---

## 3.9 Adjust Slice

```text
Adjust Slice
= 次のProcessにおいてSliceの角度・粒度・範囲・文脈を調整すること
```

これは、Stability が不十分、または Δ が大きい場合に起こりやすい。

```text
σ_n が低い
Δ_n が大きい
Representationが不十分
```

この場合、Operator Response は次の Slice を調整する。

```text
O_{n+1} = Adjust(O_n)
```

---

## 3.10 Update Orientation

```text
Update Orientation
= 次のProcessにおいてOperator Orientationを変更すること
```

Sliceの問題が、Slice操作そのものではなく、Slice前の指向性にある場合、Orientation が更新される。

```text
B_{n+1} = Update(B_n)
```

例：

```text
見る角度を変える
粒度を変える
文脈を変える
何を重要視するかを変える
許容するΔを変える
```

---

## 3.11 Update Structure

```text
Update Structure
= 次のProcessにおいてStructureそのものを更新すること
```

これは、SliceやOrientationの調整だけでは不十分な場合に起こる。

```text
S_{n+1} = Update(S_n)
```

ただし、Structure の更新は慎重に扱うべきである。

なぜなら、Gyro Logic の中核は常に Structure から始まるため、Structure更新は Loop 内でも大きな変化を意味するからである。

---

## 3.12 Jump

```text
Jump
= 既存のOrientation / Slice / Structureの連続的調整では対応できない場合に起こる非連続的な再構成
```

Jump は、Gyro Loop における重要な Response である。

次のような場合に生じる。

```text
Voidが現れる
Δが大きすぎる
既存SliceではStabilityが成立しない
既存OrientationではStructureを捉えられない
連続的な調整では解消できない
```

ただし、ここでも：

```text
VoidがJumpする
```

のではなく、

```text
Operator ResponseがVoid / Δ / Stabilityを受けてJumpを選ぶ
```

である。

---

## 3.13 Gyro Loop と Stability over time

Gyro Loop では、Stability は単発の状態量だけでなく、時間をまたいだ系列としても現れる。

単発では：

```text
σ_n = Stability of slice-done_n
```

Loop 内では：

```text
σ_0, σ_1, σ_2, ..., σ_n
```

という系列になる。

ここから見えるのが：

```text
Stability over time
```

である。

つまり：

```text
Stability as property
= 単発のslice-doneに現れる状態量

Stability over time
= 複数ProcessにまたがるStability系列
```

Gyro Loop は、Stability over time を扱うための構造でもある。

---

## 3.14 Gyro Loop と Trajectory

Gyro Loop が反復すると、Process の列が生まれる。

```text
P_0, P_1, P_2, ..., P_n
```

この列は、Gyro Logic 的には Trajectory として扱える。

```text
Trajectory =
Gyro Processの時間的系列
```

または：

```text
Trajectory =
S, B, O, X, Δ, σ, R の連続的変化の軌跡
```

ただし、この段階では Trajectory を中心概念にしすぎない。

今回の主題はあくまで：

```text
Gyro Loop = Processの反復構造
```

である。

---

## 3.15 数式風表現

Gyro Process を：

```text
P_n = (S_n, B_n, O_n, X_n, Δ_n, σ_n, R_n)
```

と置くと、Gyro Loop は：

```text
P_{n+1} = L(P_n)
```

または、より分解して：

```text
X_n + Δ_n = O_{B_n}(S_n)
σ_n = Stab(X_n, Δ_n)
R_n = Response(σ_n, Δ_n)
D_n = Decision(R_n)
```

そして：

```text
D_n ∈ {Continue, Stop, Adjust Slice, Update Orientation, Update Structure, Jump}
```

次の状態は：

```text
(S_{n+1}, B_{n+1}, O_{n+1}) = Apply(D_n, S_n, B_n, O_n)
```

となる。

Stop の場合：

```text
Loop terminates.
```

Jump の場合：

```text
(S_{n+1}, B_{n+1}, O_{n+1}) = Reconstruct(S_n, B_n, O_n)
```

と表せる。


ここでは、Gyro Logic の理論としての Loop だけを扱う。

## 3.17 誤解防止

### 誤り1

```text
Gyro LoopがStructure → Slice → Stabilityを置き換える
```

これは不正確である。

正しくは：

```text
Gyro LoopはGyro Processの反復構造であり、
Gyro Unitを置き換えない。
```

### 誤り2

```text
Stabilityが次のSliceを決める
```

これは不正確である。

正しくは：

```text
Operator ResponseがStabilityを受けて次のSliceを決める。
```

### 誤り3

```text
Gyro Loop = Gyro Process
```

これは不正確である。

正しくは：

```text
Gyro Process = 一周期
Gyro Loop = Processの反復構造
```

### 誤り4

```text
JumpはStabilityが起こす
```

これは不正確である。

正しくは：

```text
JumpはOperator Responseとして選ばれる。
```

## 3.18 正式定義として使える文章

### English

```text
Gyro Loop is the iterative structure formed when Gyro Processes are connected through Operator Response.

A Gyro Process is a single time-including operational cycle:

Structure
→ Operator Orientation
→ slice-ing
→ slice-done
→ Stability
→ Operator Response

Gyro Loop is formed when the Operator Response of one process determines whether the system continues, stops, adjusts the Slice, updates the Orientation, updates the Structure, or performs a Jump.

Gyro Loop does not replace the core principle of Gyro Logic:

Structure → Slice → Stability

Rather, it is the iterative extension of Gyro Process, which itself is the time-including unfolding of Gyro Unit.
```

### 日本語

```text
Gyro Loopとは、Gyro ProcessがOperator Responseを通じて接続されることで成立する反復構造である。

Gyro Processは、次の時間ありの一周期である。

Structure
→ Operator Orientation
→ slice-ing
→ slice-done
→ Stability
→ Operator Response

Gyro Loopは、あるProcessのOperator Responseが、継続、停止、Slice調整、Orientation更新、Structure更新、Jumpのいずれかを選ぶことで、次のProcessまたは終了へ接続される構造である。

Gyro Loopは、Gyro Logicの中核原理である

Structure → Slice → Stability

を置き換えない。

Gyro Loopは、Gyro Processの反復的拡張であり、Gyro ProcessはGyro Unitの時間ありの展開である。
```
